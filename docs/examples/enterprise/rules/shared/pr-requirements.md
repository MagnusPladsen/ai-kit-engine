# Pull Request Requirements

- Every PR must have a clear title in the format `TICKET-XX: Short description`
- PR description must include: what changed, why, and how to test
- At least two reviewers must approve before merging
- All CI checks must pass — no merging with failures
- PRs should be small and focused — one concern per PR
- Squash merge to keep history clean; the PR title becomes the commit message
- Draft PRs are encouraged for early feedback — prefix with `[WIP]`
- Address all review comments before requesting re-review
