# PR Conventions

- Small, focused PRs — one concern per PR
- Title format: `TICKET-XX: Short description` (or `feat:` / `fix:` prefix if no ticket)
- Description includes: what changed, why, and how to test
- All tests passing before requesting review
- Address all review comments before re-requesting
- Check for existing PR on the branch before creating a new one
